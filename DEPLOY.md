# Task Manager "Log" — running & deploying

npm-workspaces monorepo:

- `server/` — Express REST API (thin routes over a shared service layer), Drizzle + Neon Postgres.
- `app/` — Expo app (iOS + web from one codebase), Expo Router.
- `packages/shared/` — TypeScript types + design tokens shared by both.
- `design/` — imported Claude Design bundles (reference/source of truth for UI).

## Local development

```bash
npm install                       # install all workspaces
cp .env.example .env              # fill in DATABASE_URL, JWT_SECRET, OWNER_EMAIL

npm run db:migrate                # apply drizzle/*.sql to Neon
npm run db:seed                   # seed the 5 contexts (idempotent)

npm run server                    # API on http://localhost:4000
npm --workspace app run web       # Expo web on http://localhost:8081
# or: npm --workspace app run start  → scan QR with Expo Go (set EXPO_PUBLIC_API_URL to your LAN IP)
```

Auth in dev: no `RESEND_API_KEY` is required — the magic-link URL is printed to the
server console. Paste the `token=…` value into the app's sign-in screen.

## Deploy to Render (Blueprint)

`render.yaml` defines two services that both build from the repo root:

| Service | Type | Publishes |
|---------|------|-----------|
| `log-api` | Node web service | Express API, health check `/health` |
| `log-web` | Static site | `app/dist` (Expo web export) |

Steps:

1. Push the repo to GitHub, then in Render: **New → Blueprint** and pick the repo.
2. Set the secret env vars (marked `sync: false`) on **log-api**:
   - `DATABASE_URL` — Neon pooled connection string
   - `OWNER_EMAIL` — the only address allowed to sign in
   - `APP_URL` — the deployed **log-web** URL (used to build magic links)
   - `RESEND_API_KEY` — optional; without it, links are only logged (set it for real email)
   - `JWT_SECRET` is auto-generated by Render.
3. On **log-web**, set `EXPO_PUBLIC_API_URL` to the **log-api** URL (inlined at build time).
4. Deploy. `log-api`'s pre-deploy step runs migrations + the idempotent seed automatically.

Mobile builds (EAS Update / TestFlight) are out of scope for Phase 1 — the web export
covers the "use it by hand" milestone.
